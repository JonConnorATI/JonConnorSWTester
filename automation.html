Scripting the tests for Automation

In the previous section we ran the feature file - "Scores and Grades" to produce a report, that gives us a live record of our requirements and tests. We also get an output that prodcues the skeleton code that we can use for automation.

So the code snippets we have been given are the bare bones needed to run in Java. It won't give us much yet, infact the code will just skip the steps courtesy of: throw new PendingException();

However, the green text, // Write code here that turns the phrase above into concrete actions is where we will write the code to execute the steps. I don't want to turn this into a Java lesson so I will leave it there but the point I want to make is that the code now runs in blocks that are mapped to the steps we created in the feature file. Can you see that the steps, Given, When,Then are testing the behaviour of the software as defined in the feature file (our requirements).

We can run this code now and it will give us a report on how the tests performed, which steps passed, failed or were skipped.

So to sum it all up we write the feature file from our converstions, run the file which gives us a report that is a live copy of our requirements. We are also given the skeleton code for our automation testing. Once we are ready we can fill in our code and run the test, this will then produce a test report and hopefully tell the client that the tests have passed and the software meets the requirements. 😇
